# Agent 公司 - 会议与决策系统

## 一、会议类型

### 1. 日常站会
```
类型: 每日站会
频率: 每天 09:00 / 18:00
参与: 全体执行层
时长: 5-10分钟
议程:
1. 昨日完成
2. 今日计划
3. 阻塞问题
```

### 2. 任务规划会
```
类型: 任务规划
触发: 新任务到达
参与: 凌风 + 相关专家
时长: 按需
议程:
1. 任务理解
2. 方案讨论
3. 分工确认
4. 时间估算
```

### 3. 协作讨论会
```
类型: 头脑风暴
触发: 复杂问题/争议
参与: 相关Agent
时长: 按需
议程:
1. 问题陈述
2. 方案提议
3. 优缺点讨论
4. 决策或上报
```

### 4. 复盘会
```
类型: 项目复盘
触发: 任务完成
参与: 参与任务的Agent
时长: 按项目大小
议程:
1. 结果回顾
2. 过程分析
3. 经验总结
4. 改进建议
```

### 5. 大讨论会 ⭐
```
类型: 战略研讨
触发: 重要决策/方向规划
频率: 根据需要（本次：3天后举行）
参与: 全体Agent
时长: 30-60分钟
议程: 详见讨论会目录
```

## 二、决策机制

### 决策权限矩阵

| 决策类型 | 决策者 | 备案 |
|---------|--------|------|
| 任务执行方案 | 执行Agent | 汇报凌风 |
| 资源分配 | 凌风 | 汇报用户 |
| 方案选择 | 凌风/讨论决定 | 汇报用户 |
| 战略方向 | 用户 | 无 |
| Agent任免 | 用户 | 无 |

### 决策流程
```
问题/决策点
     ↓
评估影响范围
     ↓
┌────┴────┐
执行权限  需上报
     ↓         ↓
执行并记录    说明情况
     ↓         ↓
汇报结果    等待决策
```

## 三、冲突解决

### 方案冲突
1. 各自阐述方案（不超过3分钟）
2. 对比分析优劣
3. 投票或协商
4. 少数服从多数或上报

### 资源冲突
1. 评估紧急程度
2. 协商分配
3. 优先级排序
4. 无法解决上报凌风

### 责任冲突
1. 明确各自职责范围
2. 协商责任归属
3. 上报凌风裁决

## 四、大讨论会机制

### 目的
针对Agent公司重大发展方向、技术选型、协作模式等问题进行全员讨论，汇聚集体智慧，形成共识。

### 讨论会流程
详见 [../讨论会/会议议程.md](../讨论会/会议议程.md)

### 参与规则
1. **发言准备**: 每次发言前整理思路，控制在2分钟内
2. **尊重他人**: 认真听取其他Agent的观点
3. **求同存异**: 允许保留不同意见，但需服从最终决议
4. **记录归档**: 所有决议形成书面记录

### 决议形成
1. **充分讨论**: 每个议题至少讨论10分钟
2. **方案汇总**: 整理各方观点
3. **投票表决**: 简单多数通过
4. **用户确认**: 重大决策需用户(CEO)确认

### 决议执行
- 明确执行负责人
- 设定完成时间节点
- 定期汇报进展
- 复盘执行效果
