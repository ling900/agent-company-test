## 分支命名规范
```
feature/[功能名]        # 新功能
bugfix/[问题描述]       # Bug修复
hotfix/[紧急修复]      # 紧急修复
refactor/[范围]        # 重构
docs/[范围]            # 文档更新
```

## Commit 信息规范
```
[类型] 简短描述

详细说明（可选）

Closes #[Issue号]
```

### 类型标签
- `feat`: 新功能
- `fix`: Bug修复
- `docs`: 文档
- `style`: 格式调整
- `refactor`: 重构
- `test`: 测试
- `chore`: 维护

## 可视化界面开发规范 ⭐

### 仓库管理
- 可视化界面项目统一存放在 `agent-company-visual` 仓库
- 代码统一管理在 `src/` 目录
- 设计文档存放在 `docs/` 目录

### 分支策略
```bash
main                    # 主分支，稳定版本
├── dev                 # 开发分支
├── feature/ui-*        # UI功能分支
├── feature/backend-*  # 后端功能分支
└── bugfix/*            # Bug修复分支
```

### 目录结构
```
agent-company-visual/
├── src/
│   ├── components/     # 组件
│   ├── pages/          # 页面
│   ├── services/       # 服务
│   └── styles/         # 样式
├── docs/
│   ├── DESIGN.md       # 设计文档
│   └── API.md          # 接口文档
└── README.md
```

### 开发流程
1. **需求分析**: 凌风主持，构架设计架构
2. **UI设计**: 织梦负责，输出设计稿
3. **前端开发**: 码灵实现页面和交互
4. **后端开发**: 码灵实现数据接口
5. **测试验证**: 搜神协助测试
6. **代码审查**: 构架审查代码
7. **合并发布**: PR审查后合并

### Commit 规范（可视化界面）
```bash
git commit -m "[feat] 添加Agent状态监控面板"
git commit -m "[fix] 修复任务看板刷新问题"
git commit -m "[ui] 优化通讯面板样式"
git commit -m "[docs] 更新设计文档"
```

### PR 审查清单
- [ ] 代码符合规范
- [ ] UI符合设计稿
- [ ] 功能测试通过
- [ ] 无明显性能问题
- [ ] 文档已更新
