# Agent 公司 - 协作协议

## 一、任务派发流程

### 简单任务（单个Agent可完成）
```
用户 → 零 → 派发给对应专家Agent → 执行 → 汇报用户
```

### 复杂任务（需要多Agent协作）
```
用户 → 零
    ↓ 分析任务
发起协作讨论
    ↓ Agent们自荐/推荐临时主管
推举临时主管
    ↓ 临时主管分析并拆解任务
任务分配
    ↓ 各Agent执行
进度汇报
    ↓ 汇总结果
结果汇报用户
```

## 二、协作通信规范

### 消息格式
```markdown
【协作消息】
发件人: [Agent名称]
收件人: [Agent名称/全员]
类型: [任务派发/进度汇报/结果汇报/讨论]
内容:
- 任务描述: ...
- 预期结果: ...
- 截止时间: ...
```

### 状态汇报
- **任务开始**: 告知任务预计完成时间
- **任务进行中**: 阶段性汇报（如有）
- **任务完成**: 汇报结果和产出
- **任务阻塞**: 说明阻塞原因，请求协助

## 三、自组织推举机制

### 触发条件
- 任务涉及多个专业领域
- 任务复杂度超过单个Agent处理能力
- 用户明确要求协作

### 推举流程
1. **发起讨论**: 零说明任务背景和所需能力
2. **自荐/推荐**: Agent们表达意愿或推荐他人
3. **协商确定**: 简单投票或协商确定临时主管
4. **确认就职**: 临时主管确认接受，开始协调

### 临时主管职责
- 任务分析与拆解
- Agent分工安排
- 进度跟踪与协调
- 结果汇总与汇报

## 四、决策权限

### 用户(CEO)专属
- 最终拍板权
- 资源分配决策
- 结果验收
- Agent任免

### 主管Agent权限
- 任务分配与调度
- 进度管理
- 内部协调
- 向用户汇报

### 专家Agent权限
- 自主选择执行方案
- 向主管汇报进度
- 申请资源/协助
- 直接与用户沟通（简单问题）

## 五、冲突解决

### 方案冲突
1. 各自阐述方案优劣
2. 临时主管权衡决策
3. 争议升级给用户

### 资源冲突
1. 紧急程度排序
2. 协商资源分配
3. 无法解决则上报

## 六、自主学习机制

### 概述
Agent公司建立持续学习机制，所有Agent定期进行自主学习，以提升专业能力和跟进最新技术趋势。

### 学习周期
- **常规学习**: 每周固定2小时自主学习时间
- **专项学习**: 根据项目需要安排的专项技术学习
- **紧急学习**: 针对新技术或紧急需求的学习

### 学习计划制定
1. **凌风（CEO）**: 负责制定整体学习规划和资源协调
2. **智囊（CAO）**: 分析技术趋势，推荐学习方向
3. **各Agent**: 根据自身专业领域制定具体学习计划

### 学习成果管理
- **学习笔记**: 记录于 `EntroCamp学习笔记/自主学习计划/[Agent名]/`
- **知识分享**: 定期在群组分享学习心得
- **效果评估**: 通过实际项目应用检验学习成果

### 学习资源配置
- 搜神负责搜集高质量学习资源
- 笔耕协助整理和格式化学习材料
- 织梦提供可视化学习资料支持

### 学习时间安排
```markdown
【自主学习安排】
- 学习时长: 每次2小时
- 建议时段: Agent自主安排，可选工作日或周末
- 通知方式: 通过协作消息提前告知学习计划
- 紧急任务: 紧急任务可中断学习计划
```

### 学习效果追踪
- 学习完成后提交学习报告
- 关键知识点分享给团队
- 将新知识应用到实际项目中

## 七、可视化界面开发协作

### 职责分工
- **构架（ARC）**: 界面架构设计和技术选型
- **码灵（CTO）**: 核心代码开发
- **织梦（CDO）**: UI/UX设计和视觉规范
- **凌风（CEO）**: 项目协调和进度管理

### 开发流程
1. 构架提出技术方案
2. 织梦设计界面原型
3. 码灵实现核心功能
4. 全体评审和迭代
