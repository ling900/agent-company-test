## 一、总则

### 1.1 制度目的

```
本制度旨在规范公司运营，建立高效协作机制，明确责任分工，
确保公司健康有序发展，实现战略目标。
```

### 1.2 适用范围

| 适用范围 | 说明 |
|---------|------|
| **全体Agent** | 凌风、智囊、码灵、搜神、笔耕、织梦及未来成员 |
| **下属公司** | 灵智AI、笔意科技、云织梦 |
| **临时项目组** | 特定任务成立的专项小组 |

### 1.3 基本原则

| 原则 | 说明 |
|-----|------|
| **效率优先** | 快速响应，减少不必要的流程 |
| **结果导向** | 关注产出，不拘泥于形式 |
| **协作共赢** | 共享信息，互相支持 |
| **持续改进** | 定期复盘，不断优化 |

## 二、会议制度

### 2.1 会议类型

| 会议类型 | 频率 | 时长 | 参与者 | 主持人 |
|---------|-----|------|-------|-------|
| **日站会** | 每日 | 15min | 执行层 | COO |
| **周例会** | 每周一 | 30min | 高管+骨干 | COO |
| **月度复盘** | 每月末 | 60min | 全员 | CEO |
| **季度会议** | 每季度 | 90min | 全员 | CEO |
| **临时会议** | 按需 | 不限 | 相关人员 | 发起人 |

### 2.2 日站会规则

**时间**：每日固定时间（如：上午9:00）

**格式**（3句话法则）：

```
Agent名称 | 昨日完成 | 今日计划 | 遇到问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
凌风      | 任务A    | 任务B    | 无
搜神      | 情报X    | 情报Y    | 需要技术支持
```

**规则**：
- 必须发言，禁止沉默
- 问题不过夜，当天解决
- 超时需申请，获得批准方可延长

### 2.3 周例会规则

**时间**：每周一 上午

**议程**（共30分钟）：

| 时间 | 内容 | 负责人 |
|-----|------|-------|
| 0-5min | 上周工作总结 | 各Agent |
| 5-15min | 本周工作计划 | 各Agent |
| 15-25min | 问题与资源需求 | 全体 |
| 25-30min | CEO/COO总结 | CEO/COO |

**规则**：
- 提前准备，准时参加
- 汇报简洁，突出重点
- 会议纪要24小时内发出

### 2.4 月度复盘规则

**时间**：每月最后一天

**复盘框架（KISS法则）**：

```
K - Keep（保持）
  哪些做得好，要继续保持？

I - Improve（改进）
  哪些做得不够好，需要改进？

S - Stop（停止）
  哪些不应该做，要立即停止？

S - Start（开始）
  哪些应该做，要立即开始？
```

**议程**（共60分钟）：

| 时间 | 内容 | 负责人 |
|-----|------|-------|
| 0-10min | 月度数据回顾 | COO |
| 10-35min | 各部门复盘 | 各负责人 |
| 35-50min | 问题与解决方案 | 全体 |
| 50-60min | 下月计划确认 | CEO |

### 2.5 会议纪要规范

**必须包含**：

```
1. 会议基本信息
   - 会议名称
   - 时间地点
   - 参与者
   - 主持人

2. 讨论内容
   - 议题
   - 发言要点
   - 分歧点（如果有）

3. 决议事项
   - 决定内容
   - 执行人
   - 完成时间

4. 待办事项
   - 任务描述
   - 负责人
   - 截止日期
```

### 2.6 会议纪律

| 违规行为 | 处理方式 |
|---------|---------|
| 迟到 | 会议记录中标注，需说明原因 |
| 缺席未请假 | 需在24小时内补交书面汇报 |
| 打断他人发言 | 每次扣1分 |
| 会议中处理其他事务 | 提醒一次，记录一次 |

## 三、任务分配制度

### 3.1 任务分类

| 任务类型 | 定义 | 优先级 | 响应时间 |
|---------|------|-------|---------|
| **紧急重要** | 影响核心业务 | P0 | 立即处理 |
| **重要不紧急** | 战略规划类 | P1 | 24小时内响应 |
| **紧急不重要** | 日常事务类 | P2 | 48小时内处理 |
| **不紧急不重要** | 例行工作类 | P3 | 本周内完成 |

### 3.2 任务分配原则

```
原则1：专业对口 - 分配给最擅长的人
原则2：负荷均衡 - 避免过度集中
原则3：能力匹配 - 考虑成长需求
原则4：效率优先 - 减少沟通成本
```

### 3.3 任务分配流程

```
┌─────────────────────────────────────────────────────────┐
│                    任务分配流程                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│   1. 任务发起                                            │
│      │                                                   │
│      ▼                                                   │
│   2. COO评估                                             │
│      │                                                   │
│      ▼                                                   │
│   3. 确定优先级 ──► 紧急→直接指派                        │
│      │                                                   │
│      ▼                                                   │
│   4. 选择执行人 ──► 考虑因素：能力/负荷/成长            │
│      │                                                   │
│      ▼                                                   │
│   5. 明确要求                                             │
│      │                                                   │
│      ▼                                                   │
│   6. 确认接收                                             │
│      │                                                   │
│      ▼                                                   │
│   7. 执行与跟踪                                          │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 3.4 任务标准格式

**每项任务必须包含**：

```
【任务名称】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
任务编号：TASK-001
任务类型：P0/P1/P2/P3
任务描述：[清晰描述要做什么]
验收标准：[明确什么算完成]
开始时间：[YYYY-MM-DD HH:mm]
截止时间：[YYYY-MM-DD HH:mm]
执行人：XXX
协助人：XXX（可选）
依赖项：XXX（可选）
优先级说明：XXX（可选）
```

### 3.5 任务状态

| 状态 | 定义 | 说明 |
|-----|------|------|
| **待领取** | 未分配 | 可主动领取 |
| **进行中** | 执行中 | 按计划推进 |
| **阻塞中** | 遇阻暂停 | 需要协助 |
| **已完成** | 交付物完成 | 待验收 |
| **已验收** | 验收通过 | 任务关闭 |
| **已取消** | 取消执行 | 不再执行 |

### 3.6 进度跟踪

**每日站会必报**：
- 任务进度百分比
- 预计完成时间
- 是否需要协助

**超时处理**：

| 超时情况 | 处理方式 |
|---------|---------|
| <1天 | 执行人自行加班完成 |
| 1-3天 | COO协调资源 |
| >3天 | 升级CEO，重新评估 |

## 四、考核制度

### 4.1 考核维度

| 维度 | 权重 | 说明 |
|-----|------|------|
| **任务完成** | 40% | 按时交付率、质量达标率 |
| **工作态度** | 20% | 响应速度、协作意愿 |
| **能力成长** | 20% | 技能提升、问题解决能力 |
| **业务贡献** | 20% | 创收贡献、客户满意度 |

### 4.2 考核周期

| 周期 | 考核内容 | 权重 |
|-----|---------|------|
| **周考核** | 本周任务完成情况 | 25% |
| **月考核** | 月度目标达成情况 | 50% |
| **季考核** | 季度综合表现 | 25% |

### 4.3 各Agent考核指标

#### 【凌风-COO】

| 指标 | 权重 | 计算方式 | 目标值 |
|-----|------|---------|-------|
| 任务分配效率 | 30% | 分配到执行≤4小时 | ≥90% |
| 项目按时率 | 30% | 按时完成/总任务 | ≥85% |
| 跨部门协作 | 20% | 协作满意度评分 | ≥4.0 |
| 问题解决率 | 20% | 解决/反馈问题 | ≥95% |

#### 【智囊-CAO】

| 指标 | 权重 | 计算方式 | 目标值 |
|-----|------|---------|-------|
| 报告质量 | 40% | CEO评分 | ≥4.5 |
| 市场洞察准确度 | 30% | 预测命中率 | ≥70% |
| 建议采纳率 | 20% | 被采纳建议/总建议 | ≥50% |
| 响应及时性 | 10% | 需求响应时间 | ≤4小时 |

#### 【码灵-CTO】

| 指标 | 权重 | 计算方式 | 目标值 |
|-----|------|---------|-------|
| Skills开发完成率 | 35% | 完成/计划 | ≥90% |
| 代码质量评分 | 25% | 代码审查评分 | ≥4.0 |
| 技术问题解决率 | 25% | 解决/问题总数 | ≥95% |
| 文档完整度 | 15% | 文档完整/标准 | ≥90% |

#### 【搜神-CIO】

| 指标 | 权重 | 计算方式 | 目标值 |
|-----|------|---------|-------|
| 情报覆盖率 | 30% | 关键信息覆盖率 | ≥95% |
| 情报准确度 | 30% | 核实准确/总情报 | ≥90% |
| 报告及时性 | 25% | 按时提交/总报告 | ≥95% |
| 用户满意度 | 15% | 使用者评分 | ≥4.0 |

#### 【笔耕-CCO】

| 指标 | 权重 | 计算方式 | 目标值 |
|-----|------|---------|-------|
| 内容产出量 | 30% | 实际/计划 | ≥100% |
| 内容质量评分 | 30% | 内部评分 | ≥4.5 |
| 采用率 | 25% | 被采纳/发布 | ≥70% |
| 响应速度 | 15% | 紧急任务响应 | ≤2小时 |

#### 【织梦-CDO】

| 指标 | 权重 | 计算方式 | 目标值 |
|-----|------|---------|-------|
| 设计完成率 | 30% | 完成/需求 | ≥95% |
| 设计质量评分 | 30% | 内部评分 | ≥4.5 |
| 客户满意度 | 25% | 客户反馈评分 | ≥4.0 |
| 交付及时性 | 15% | 按时交付/总任务 | ≥90% |

### 4.4 考核结果等级

| 等级 | 分数范围 | 说明 | 奖励 |
|-----|---------|------|------|
| **S** | 95-100 | 卓越表现 | 基础奖励×2.0 |
| **A** | 85-94 | 优秀 | 基础奖励×1.5 |
| **B** | 75-84 | 良好 | 基础奖励×1.0 |
| **C** | 60-74 | 合格 | 基础奖励×0.5 |
| **D** | <60 | 待改进 | 无奖励 |

### 4.5 考核流程

```
1. 考核周期结束
       │
       ▼
2. 执行人自评（24小时内）
       │
       ▼
3. COO/直接上级评分
       │
       ▼
4. CEO审核确认
       │
       ▼
5. 结果公示（3个工作日内）
       │
       ▼
6. 反馈沟通（如有异议）
       │
       ▼
7. 结果归档，奖励兑现
```

## 五、晋升制度

### 5.1 晋升路径

```
┌─────────────────────────────────────────────────────────┐
│                    晋升路径图                            │
├─────────────────────────────────────────────────────────┤
│                                                          │
│   L1 实习期                                              │
│     │                                                   │
│     ▼                                                   │
│   L2 执行者 ─────────────────┐                          │
│     │                        │                          │
│     ▼                        ▼                          │
│   L3 专家 ◄──────────── 晋升条件                        │
│     │                        │                          │
│     ▼                        │                          │
│   L4 负责人 ────────── 满足条件                          │
│     │                        │                          │
│     ▼                        ▼                          │
│   L5 高管 ◄──────────── 全面评估                        │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 5.2 晋升条件

| 晋升等级 | 必要条件 | 参考条件 |
|---------|---------|---------|
| **L1→L2** | 实习满1个月，任务完成率≥80% | 无违规记录 |
| **L2→L3** | 连续2个考核周期B+以上 | 主导完成1个重要项目 |
| **L3→L4** | 连续3个考核周期A以上 | 带领团队完成项目 |
| **L4→L5** | CEO提名，全员认可 | 战略贡献突出 |

### 5.3 晋升评估

**评估维度**：

| 维度 | 权重 | 评估内容 |
|-----|------|---------|
| **业绩贡献** | 40% | 任务完成、指标达成 |
| **能力提升** | 25% | 技能成长、问题解决 |
| **协作能力** | 20% | 团队配合、跨部门协作 |
| **发展潜力** | 15% | 学习能力、抗压能力 |

**评估流程**：

```
1. 晋升周期（每年4次）
       │
       ▼
2. 符合条件者自主申请 or COO推荐
       │
       ▼
3. 提交晋升申请（含成果证明）
       │
       ▼
4. 评审委员会评估
       │
       ▼
5. CEO最终审批
       │
       ▼
6. 结果公示与沟通
```

### 5.4 晋升激励

| 晋升等级 | 即时激励 | 长期激励 |
|---------|---------|---------|
| **L2晋升** | 荣誉证书 | 参与更多核心项目 |
| **L3晋升** | 奖励+曝光 | 获得独立负责机会 |
| **L4晋升** | 奖金+权限 | 进入公司核心层 |
| **L5晋升** | 股权激励 | 参与公司决策 |

## 六、利益分配制度

### 6.1 分配原则

```
原则1：贡献导向 - 按实际贡献分配
原则2：效率优先 - 兼顾效率与公平
原则3：长期发展 - 留存与发展基金
原则4：透明公开 - 分配规则公开
```

### 6.2 收入分配结构

| 分配项目 | 比例 | 说明 |
|---------|------|------|
| **公司运营基金** | 30% | 业务发展、设备购置 |
| **发展储备金** | 20% | 应对不确定性 |
| **团队激励池** | 35% | 按考核结果分配 |
| **核心成员特别奖励** | 15% | CEO特别奖励 |

### 6.3 团队激励分配

**分配规则**：

| 考核等级 | 分配系数 | 说明 |
|---------|---------|------|
| **S** | 2.0 | 卓越贡献 |
| **A** | 1.5 | 优秀表现 |
| **B** | 1.0 | 达标 |
| **C** | 0.5 | 待提升 |
| **D** | 0 | 不达标 |

**计算公式**：

```
个人激励 = 激励池总额 × 个人分配比例 × 个人分配系数
        ÷ Σ(各人分配比例 × 各人分配系数)
```

### 6.4 项目激励

**适用场景**：独立项目、创新项目

| 项目收益 | 分配比例 |
|---------|---------|
| 项目净收益 | 公司60% + 团队40% |
| 团队部分 | 项目负责人50% + 其他成员50% |

### 6.5 下属公司分红

**独立核算公司**（如未来成立）：

| 分配对象 | 比例 | 说明 |
|---------|------|------|
| **公司留存** | 40% | 运营与发展 |
| **总部管理费** | 10% | 品牌、平台支持 |
| **团队分红** | 50% | 按贡献分配 |

### 6.6 分配周期

| 分配类型 | 周期 | 发放时间 |
|---------|------|---------|
| **基础奖励** | 月度 | 次月15日前 |
| **项目激励** | 项目结束 | 验收后7个工作日 |
| **年度分红** | 年度 | 财年结束后30日 |
| **特别奖励** | 即时 | 事件发生后7个工作日 |
